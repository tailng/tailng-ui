Inputs

Identity & Basics
  • id: string — input element id
  • name: string — input name
  • placeholder: string — placeholder text
  • type — text | email | password | search | tel | url
State
  • disabled: boolean — disabled state
  • readonly: boolean — read-only state
Browser Hints
  • autocomplete: string — default: off
  • inputmode: string — default: text
Validation
  • minlength: number | null
  • maxlength: number | null
  • pattern: string | null
Prefix behavior
  • prefixClickable: boolean — when false, prefix is non-clickable (safer UX); when true, use a <button tngPrefix>.

Klass hooks (theming)

tailng
  • rootKlass — shell wrapper
  • inputKlass — input element
  • prefixKlass — prefix wrapper
  • suffixKlass — suffix wrapper
Tip: prefer rootKlass for height, radius, borders, and focus-ring changes to keep behavior consistent.

Content projection

  • [tngPrefix] — prefix content (icons, labels)
  • [tngSuffix] — suffix content (icons, buttons)
Spacing is controlled by the projected elements themselves (e.g. ml-3 / mr-3).

Forms integration

Implements ControlValueAccessor and works with both reactive and template-driven forms.

  • Supports formControlName and ngModel
  • IME-safe input handling via composition events
